# Агенты

## 1. EDA-агент

Делает первичный анализ датасета.

**Важно:** работает только в режиме чтения и не меняет исходный датасет.

**Что делает:**
- описывает колонки;
- определяет типы данных;
- находит пропуски, дубликаты и другие проблемы;
- формирует список проблем по датасету.

---

## 2. Агент очистки данных

Устраняет одну конкретную проблему из списка EDA-агента.

**Что делает:**
- принимает на вход одну проблему;
- исправляет только её;
- сохраняет результат в новый CSV-файл;
- кратко описывает, что было изменено.

---

## 3. Агент моделирования

Отвечает за выбор ML-подхода.

**Что делает:**
- предлагает возможные таргеты;
- определяет тип задачи;
- выбирает модели для обучения;
- предлагает метрики качества.

---

## 4. Агент обучения модели

Обучает одну выбранную модель.

**Что делает:**
- принимает выбранную модель и таргет;
- обучает модель;
- считает метрики;
- сохраняет результат обучения.

---

## 5. Агент отчетности

Формирует финальный отчет по результатам работы пайплайна.

**Что делает:**
- собирает результаты предыдущих агентов;
- пишет summary;
- описывает качество данных;
- кратко сравнивает результаты моделей;
- формирует выводы и рекомендации.
